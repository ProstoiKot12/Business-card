# Подготовка и Развертывание

В этом разделе описано, как настроить окружение, переменные и запустить бота в различных режимах.

## Конфигурация (.env)

Все настройки проекта управляются через переменные окружения. Для начала скопируйте файл `.env.example` в `.env`:

```bash
cp .env.example .env
```

### Основные переменные

- `BOT_TOKEN`: Токен вашего Telegram-бота, полученный у [@BotFather](https://t.me/BotFather).
- `DATABASE_URL`: Строка подключения к PostgreSQL (формат: `postgresql+asyncpg://user:pass@host:port/dbname`).
- `REDIS_URL`: Строка подключения к Redis.
- `ADMIN_IDS`: Список Telegram ID пользователей, которым доступна админ-панель (команда `/admin`). Через запятую, например `123456789,987654321`.
- `OWNER_ID`: Telegram ID владельца портфолио (используется для ссылок).
- `OWNER_NAME`: Имя владельца (будет отображаться в боте).
- `OWNER_TELEGRAM_URL`, `OWNER_GITHUB_URL`, `OWNER_MAX_URL`: Ссылки на ваши профили.
- `MAIN_PHOTO_FILE_ID`: (Опционально) `file_id` главной фотографии, которую бот будет отправлять при старте. Это значение можно получить, отправив фото боту и посмотрев логи.
- `LOG_LEVEL`: Уровень логирования (например, `INFO`, `DEBUG`).

## Локальный запуск для разработки (Dev-режим)

В dev-режиме бот запускается локально из виртуального окружения (venv), а базы данных (PostgreSQL и Redis) поднимаются в Docker.
Для удобства предусмотрен PowerShell-скрипт `scripts/dev.ps1`.

### Шаги:

1. Убедитесь, что у вас установлен **Python 3.11+** и запущен **Docker Desktop** (или Docker Engine).
2. Настройте файл `.env`.
3. Запустите скрипт:

```powershell
.\scripts\dev.ps1
```

**Что делает скрипт по умолчанию?**
- Создает виртуальное окружение `.venv` (если его нет).
- Устанавливает зависимости из `pyproject.toml` (включая dev-зависимости), если они изменились.
- Поднимает контейнеры `postgres` и `redis` через `docker-compose.yml`.
- Применяет миграции БД Alembic (если есть новые миграции).
- Запускает бота локально командой `python -m app.main`.

**Дополнительные флаги скрипта:**
- Принудительная переустановка зависимостей: `.\scripts\dev.ps1 -Install`
- Принудительный прогон миграций: `.\scripts\dev.ps1 -Migrate`

## Полностью в Docker (Production)

Для деплоя на сервер (VPS) рекомендуется запускать и бота, и базы данных внутри Docker.

Для этого в проекте есть два compose-файла:
- `docker-compose.yml` (Базы данных: PostgreSQL и Redis)
- `docker-compose.bot.yml` (Сам Telegram-бот)

### Запуск:

```bash
docker compose -f docker-compose.yml -f docker-compose.bot.yml up --build -d
```

> [!TIP]
> Флаг `-d` запустит контейнеры в фоновом режиме (detached mode). Вы можете просматривать логи с помощью команды `docker compose logs -f bot`.

### Применение миграций в Docker

При старте контейнера с ботом он не накатывает миграции автоматически (обычно это плохая практика в production). Чтобы применить миграции внутри работающего контейнера, выполните:

```bash
docker compose -f docker-compose.yml -f docker-compose.bot.yml exec bot alembic upgrade head
```
